## 验收结论

任务七的软件目标已通过：模拟主板能够稳定输出当前初中组三协议的 `hello / telemetry / event / ack`，能够复现 A 键、GPIO17 旋钮、水浸 P1、小暑与大寒策略，并按命令 ID 幂等返回 ACK。

## 本次实现

- `tools/n16r8_mock_board.py`
  - 固定身份：`smartlife-junior / smartlife-junior-solar-home-v1 / n16r8_esp32s3`。
  - 采用可重复的逻辑时钟和递增 `seq`，无随机传感器值。
  - 支持当前命令白名单、16 项 ACK 缓存、`lastAppliedCommandId`、P1 安全锁和延迟模式命令。
  - 支持有限运行的 `steady`、`water-demo`、`controls-demo` 场景，以及从 stdin 接收命令的 `--serve` 模式。
  - `3500ms` 内（含边界）视为新鲜帧，超过 `3500ms` 后可供网页任务验证离线状态。
- `tools/capture_serial_evidence.py`
  - 原始 JSONL 逐行原样保存，不给帧增加时间字段。
  - 时间、行号、字节数与 SHA-256 写入独立 `.timestamps.jsonl` 旁车文件。
- `tools/test_mock_board.py`
  - 覆盖身份帧、序号、ACK 闭环、本地事件、P1 恢复、重复命令、安全锁、离线边界和采集完整性。
