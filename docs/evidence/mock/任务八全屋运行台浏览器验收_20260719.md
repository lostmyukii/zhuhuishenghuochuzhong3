## 浏览器检查

本地预览地址：`http://127.0.0.1:8767/dashboard/`。验收完成后已停止临时服务。

1. 初始页面不填充正常值，显示“串口未连接 / 等待实时数据”。
2. 注入任务七 `source=mock` hello 后，顶部持续显示“模拟主板 / 模拟数据 / 不代表实物动作”，不会升级为真板在线。
3. 注入水浸遥测后，卫生间房间变红；通知显示 `GPIO8`、水浸原因以及风扇 100%、RGB 红、低压灯 GPIO 目标关闭、蜂鸣器目标报警。
4. 模拟点击 Sleep 后，页面接收同 ID ACK 和更新后的 telemetry，才显示“状态已更新”；只收到 ACK 的纯逻辑测试停留在“已接受，等待状态更新”。
5. 停止模拟遥测超过 3500ms 后，顶部显示“开发板离线”，实时数据和告警结论清空为等待状态。
6. 点击断开后立即清空实时结论，不残留 3500ms 的旧在线状态。
7. Chromium 控制台错误与警告均为 0。
8. `390px` 视口实测：`innerWidth=390`、`documentElement.scrollWidth=390`、`body.scrollWidth=390`。

## 真实性边界

- Web Serial 代码已在支持 `navigator.serial` 的本地 Chromium 环境加载，但本次没有选择真实 CH340/N16R8 串口。
- “串口已连接”与“开发板在线”是两个状态；只有匹配项目 hello 且帧未过期才能显示真板在线。
- 红色告警只来自 `telemetry.alerts`，不会根据 PIR、水浸或 MQ2 原始值在浏览器自行生成。
- 继电器只写“GPIO 目标开/关”，不宣称触点已经动作。
- 本次没有烧录、没有真串口帧、没有实体执行器证据，也没有公网部署。
- 任务九语音和任务十完整 G1 汇总尚未执行，因此项目当前仍低于 `mock-passed`。
